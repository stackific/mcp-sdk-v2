The value type behind each `requestedSchema.properties` entry: the union of the
four primitive field schemas a form may use. (§20.4)

```ts
PrimitiveSchemaDefinition = StringSchema | NumberSchema | BooleanSchema | EnumSchema
```

Order matters: enum members (which carry `enum`/`oneOf`/`items`) are tried
before the bare [StringSchemaSchema](StringSchemaSchema.md) so a string-typed enum is not
mis-matched as a free-text string. [NumberSchemaSchema](NumberSchemaSchema.md) and
[BooleanSchemaSchema](BooleanSchemaSchema.md) are unambiguous by `type`. Use
[classifyPrimitiveSchema](../functions/classifyPrimitiveSchema.md) for the precise selected member.
